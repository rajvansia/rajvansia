Pneumonia Clinical Stability Predictor Using FHIR & React

I started an online masters in Computer science at Georgia tech this past fall. OMSC program is great so far as its affordable and because its all online it fits around my schedule. One of the classes I took this semester was healthcare informatics. We had a cool class project where we would use FHIR to create an app that solves a specific healthcare problem. This was a great opportunity to be able to work on a real world problem with a physician from Emory.  Our objective was to predict the clinical stability of patients with clinically acquired pneumonia. I was in charge of the frontend and in this post I will show you some of the react elements I used for the UI.

We created a rules based algorithm that determines the stability based on a 24 hour sliding window of vital signs data. This is done using our python flask backend.  This allows patients from a FHIR server that are diagnosed with pneumonia to be exposed through API.  Here you can find a sample payload from our API.  Some of the inspiration came from the real-time medication insights for CDS hooks.

Front End Layout

We utilized a card component to discreetly . The view of the dashboard lists all of individual cards with patent information. Its a simple concept where a list of patient objects are mapped to individual cards that displays their own vitals data.  It displays the most pertinent data such as stability prediction, length of stay and vitals signs data.

Redux Saga Data Binding

Our api payload is a list of patient objects that have pneumonia. Within the object there are vital signs and demographic data. We will load the data on component mount. If there is no data in our object yet we will display that the app is loading.

Styling

Inline conditional formatting based on the current state. In this case if the status of the patient is stable the text is displayed as green.
